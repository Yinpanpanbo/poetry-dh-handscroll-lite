# 一句话介绍

我们用基础 HTML、CSS 和 JavaScript，把王维《山居秋暝》做成一卷可以展开、定位和解释的双语数字手卷。简化版只保留一个核心交互和四项可讲清的前端技术。

![英文主页面](images/simplified/01-home-desktop.png){ width=100% }

# 为什么简化

旧版包含语法地图、接受史、矿物材料和多组文化证据，内容完整但课堂讲解线路过长。新版本没有降低实现质量，而是删除非核心展示，让每个保留模块都服务于同一个问题：**怎样让八行诗从“空”逐步移动到“留”？**

页面现在只有四段：封面、核心手卷、技术卡片、来源。演示时约 3—5 分钟即可走完。

# 文件结构

```text
index.html      英文主页面：结构与双语内容
zh.html         中文辅助页面：同一交互的中文界面
style.css       共享视觉、布局、动画与响应式规则
app.js          共享六站数据与交互状态
assets/         原创 SVG 长卷
```

两个语言页面共享 CSS 和 JavaScript，避免维护两套不同功能。部署产物也只有这五项。

# 技术一：语义 HTML 组织意义

```html
<article class="scene">
  <div class="verse">
    <div lang="zh-CN"><b>明月松间照</b></div>
    <p>The bright moon shines between the pines.</p>
  </div>
</article>
```

`section` 区分页面章节，`article` 表示六个诗画场景，`lang` 标记双语文本，真正的 `button` 与 `input type="range"` 提供键盘和无障碍基础。HTML 不负责动画，只建立可理解的内容骨架。

# 技术二：CSS 构成手卷与响应式布局

```css
.handscroll { display: flex; overflow-x: auto; }
.scene { flex: 0 0 100%; }
```

Flexbox 把六幕排成一条横卷，每幕恰好占一个画心宽度；Grid 把画心和卷外题跋放在同一展陈框架中。SVG 作为连续背景横跨六幕，诗句则叠加在各自停驻点。

媒体查询把桌面端的左右结构改为手机端上下结构，六站按钮允许横向浏览。颜色变量统一深墨绿、绢色和泥金，修改变量就能整体调整视觉系统。

# 技术三：CSS 只动画一个有意义的状态

```css
.scroll-preview figure { clip-path: inset(0 44%); }
.hero.opened figure { clip-path: inset(0); }
```

点击启卷时，JavaScript 只给首页添加 `.opened`；具体的展开宽度、时间和缓动都由 CSS 完成。这说明“JavaScript 改状态，CSS 负责表现”。`prefers-reduced-motion` 会为需要减少动态的用户关闭过渡。

# 技术四：JavaScript 同步一个状态

```js
function goTo(index) {
  handscroll.scrollTo({ left: scenes[index].offsetLeft,
                        behavior: 'smooth' });
}
```

程序只维护两个核心变量：当前停驻点 `current` 和当前解释视角 `view`。一次位置变化会同步更新：

- 长卷位置；
- `01 / 06` 进度；
- 当前六站按钮；
- 右侧题跋；
- 连续范围条。

Read、Visualise、Boundary 的内容保存在同一站的数据对象中，因此切换解释不需要复制三套 HTML。英文和中文数据也由页面的 `lang` 属性自动选择。

# 内容怎样服务交互

六站不是把八行诗机械切开，而是六个可感知的变化：雨后环境、月光、清泉、竹中人声、莲动舟行、留下的判断。读者从自然景观逐步发现声音和人物，最后理解“空”不是无人，而是一种不被喧闹占满、仍可生活的空间。

原创 SVG 对应这六个变化，但页面明确标注它是当代视觉转译，不是历史复原。Boundary 视角把这种限制直接呈现给用户。

# 部署与验证

项目不需要 npm、数据库或服务器。GitHub Actions 把 HTML、CSS、JavaScript 和 SVG 复制到 Pages 产物，因此仓库链接可以直接转成公开网页。

本机 Chrome 已检查桌面和手机尺寸；六站定位、解释切换、进度条、启卷、移动菜单、中英文数据和键盘方向均可运行，页面无横向溢出。

# 课堂讲解顺序

1. 首页：一句话介绍问题与启卷动画。
2. 手卷：点击三个代表站，说明“一屏一景一诗句”。
3. 解释层：在同一站切换三种视角，说明数组数据。
4. 技术卡：按 HTML、CSS、Motion、JavaScript 回到源码。
5. 手机页：说明媒体查询与共享代码。

结尾可以概括：项目没有靠堆叠框架制造复杂感，而是让少量基础技术共同服务一个清楚的阅读动作。
